# WarmCall Design Principles

**目标：80 岁老人第一次使用，30 秒内知道怎么操作。**

## 核心原则

### 1. 尽量不用键盘
- 老人端优先使用**大按钮、图标、语音**（或语音模拟）
- 邀请码使用 **4 位纯数字**，大字号独立输入框
- 提供「快捷话术」按钮，一键发送常见句子

### 2. 能点图标就不要读长文字
- 入口页用超大 emoji 图标：👵 我是老人 / 👨‍👩‍👧 我是家人
- 老人首页三个按钮均带图标：🎤 开始聊天 / 👨‍👩‍👧 看看家人 / ❤️ 今日心情
- 文字简短、口语化，避免专业术语

### 3. 一个页面最多 3 个主要按钮
- 老人首页严格限制为 3 个主操作
- 次要操作（如退出）使用小字链接，不占主视觉

### 4. 字体大、按钮大
- 老人端正文 ≥ 1.2rem，标题 ≥ 1.75rem
- 主按钮最小高度 52px，首页卡片按钮 ≥ 120px
- 邀请码显示 ≥ 3.5rem，加宽字距

### 5. 流程固定
```
欢迎页 → 输入 4 位数字 → 老人首页 → 选功能 → 完成提示 → 回首页
```
- 不跳转子女后台
- 每步有明确「返回」「回首页」

### 6. 操作完成后必须有明确提示
- 聊天结束：「聊完了，真好」+ 回首页按钮
- 心情记录：「记下了，家人会知道」
- 绑定失败：「数字不对，请让家人再告诉您一次」

## 页面规范

| 页面 | 主操作数 | 键盘 | 说明 |
|------|---------|------|------|
| 欢迎页 `/` | 2 | 否 | 两个大卡片 |
| 老人绑定 `/elder/bind` | 1 | 仅 4 位数字 | 数字键盘 |
| 老人首页 `/elder/home` | 3 | 否 | 固定三按钮 |
| 老人聊天 `/elder/chat` | 1+快捷 | 否 | 按住说话 + 快捷语 |
| 子女端 `/family` | 不限 | 是 | 完整管理功能 |

## 色彩与氛围
- 暖橙主色 `#e07a3a`，米白背景
- 圆角 ≥ 16px，卡片有轻柔阴影
- 语气温暖、不吓人，避免红色警告式文案

## 子女端差异
- 可保留完整表单、列表、摘要详情
- 邀请码区域突出：「请告诉老人输入这个 4 位数字：4286」
- 提供复制、重新生成邀请码

## 技术预留
- `client/src/services/voiceService.js`：语音识别 / 合成接口
- 当前 Alpha：`mockVoiceTranscribe()` 模拟按住说话

## 验收清单
- [ ] 老人不看说明能点进「我是老人」
- [ ] 4 位数字输入框足够大，只接受数字
- [ ] 老人首页恰好 3 个大按钮
- [ ] 聊天页无文字输入框（或极弱化）
- [ ] 每步结束有明确反馈
- [ ] 子女端可见 4 位邀请码并可复制
